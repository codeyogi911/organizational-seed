# Seed roadmap

The pattern evolves only by folding back what live instances prove. Versions
are themes, not dates.

| Version | Theme | What ships |
|---|---|---|
| **v0.3 — Verify** ✅ *shipped* | checks become universal | `tools/doctor` (immune surveillance: links, kind frontmatter, ledger validity, lease freshness, unruled "applied" proposals, CODEOWNERS presence), the mechanical conserved-file gate ([docs/enforcement.md](enforcement.md) + `CODEOWNERS.example`), and the design vocabulary made explicit ([docs/biomimicry.md](biomimicry.md)). Continuing: per-process check generalization, kind→JSON-schema compilation (v0.5) |
| **v0.4 — Many hands** | multi-operator maturity | the weekly review as a core process (Now compaction, ledger batch-review, deadman grid, pattern-sync duty), lease heartbeat helper, per-operator-class credential doctrine |
| **v0.5 — Hiring library** | forkable stewards | reference charters (books, support, audit) and domain canon packs — hiring an agent accountant by forking a file |
| **v0.6 — Always on** | headless operation | runner-independent scheduled Mounts (cloud/CI; no laptop-lid dependency), deadman escalation to the Founder's phone, CLI→MCP transport evolution |
| **v1.0 — The network** | many instances | `seed init` bootstrap ritual, holding-org pattern for multiple businesses, cross-instance lesson flow, docs site |

Standing rule: pattern-level lessons flow here from live instances at their
weekly reviews; instance data never does.
