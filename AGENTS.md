# For any coding agent entering this repo

You are entering an **organization, not a codebase**. This repo is durable
organizational state kept as plain files.

1. Read [ORG.md](ORG.md) first — it tells you the purpose, roles, current state,
   and next action.
2. Obey [AUTHORITY.md](AUTHORITY.md). Access to a tool never implies permission to
   use it. Never touch a conserved file without an approved Proposal.
3. Do work as Tasks under `processes/`; cite Records as evidence for every claim.
4. This file and everything harness-specific (`.claude/`, `CLAUDE.md`) are
   disposable Mounts — never store organizational state in them.
